## ğŸ¯ Key Architectural Decisions

### 1. **WalletConnect v2 Standalone** (Not Web3Modal v3)
**Why**: Simpler, more stable, smaller bundle, proven pattern

### 2. **Ethers.js v5** (Not v6)
**Why**: Better browser support, simpler API, more stable

### 3. **Vanilla JavaScript** (No React/Vue)
**Why**: Works with existing Flask/Jinja2 templates, no build step

### 4. **Class-Based Architecture**
**Why**: Encapsulation, testability, clear API surface

### 5. **Persistent Connection via localStorage**
**Why**: Better UX, but validated against actual session

### 6. **Network Validation on Every Transaction**
**Why**: Fail fast, clear error messages, better UX

### 7. **Events-Based UI Updates**
**Why**: Reactive, handles all edge cases (account switch, network switch)
